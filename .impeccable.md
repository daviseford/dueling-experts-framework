## Design Context

### Users

Solo developers and power-user senior engineers running DEF on their own projects. They kick off agent sessions and want a glanceable heads-up display while agents debate and implement — not an interactive chat. They may leave it open in a second monitor or tab and check back periodically. The UI should reward both sustained attention (reading the debate) and quick glances (status at a glance).

### Brand Personality

**Clean, Calm, Professional.** DEF is an understated, polished developer tool. It stays out of the way and lets the structured debate content speak for itself. No flashiness, no gimmicks — just clarity and confidence. The "dueling experts" concept is expressed through structure (turn cards, agent identity colors, phase progression) rather than visual spectacle.

### Emotional Goal

**Confidence and trust.** The interface should make the developer feel: "I can see exactly what's happening and it's under control." Transparency breeds trust — every status, cost, decision, and phase transition should be visible and legible at a glance.

### Aesthetic Direction

**Mission control.** Think Vercel dashboard or Linear — status indicators, data density, glanceable metrics. Information-rich but never cluttered. Cards, badges, and subtle color coding convey multiple data dimensions simultaneously. The UI is a monitoring surface, not a conversation interface.

**Reference points:** Vercel (deployment status), Linear (issue tracking density), Raycast (clean developer chrome).

**Anti-references:**
- ChatGPT-style chat UI — this is a structured debate with phases and decisions, not a casual conversation
- Enterprise dashboards — no heavy chrome, sidebar nav trees, or overwhelming data grids
- Gamer/neon aesthetic — no glowing neon, cyberpunk gradients, or over-the-top visual effects

### Design System (Existing)

- **Framework:** React 19, Tailwind v4, shadcn/Radix primitives, Vite
- **Typography:** Sora (sans-serif body), JetBrains Mono (code/metadata)
- **Color space:** OKLch for perceptual uniformity across light/dark
- **Theme:** Light + dark mode via next-themes, system default
- **Icons:** Lucide React, 16px default
- **Border radius:** 0.625rem base (10px), scaled via CSS calc
- **Semantic colors:**
  - Agents: Blue (Claude), Emerald (Codex), Violet (Human), Slate (System)
  - Phases: Orange (Plan), Lime (Debate), Cyan (Implement), Pink (Review)
  - Status: Emerald (active), Amber (paused), Muted (completed), Red (interrupted)
- **Animations:** Thinking glow, slide-up, fade-in, scan gradient — all respect `prefers-reduced-motion`
- **Accessibility:** Reduced-motion support, semantic HTML, high-contrast color pairs, icons paired with labels

### Design Principles

1. **Transparency over abstraction.** Show the real state — turns, costs, decisions, phase, agent identity. Don't hide complexity; organize it.
2. **Glanceable density.** Every pixel should earn its place. Badges, left-border accents, and status indicators pack meaning into small space without feeling cramped.
3. **Calm confidence.** Muted palettes, subtle depth (soft shadows, semi-transparent borders), no jarring transitions. The UI should feel steady even when agents are actively working.
4. **Content is king.** The debate prose, decisions, and diffs are the primary content. Chrome exists to frame and organize, never to compete for attention.
5. **Semantic color, not decorative color.** Color always signals meaning (agent identity, phase, status). Avoid color for pure decoration.
